# Sustainable OS — 운영 원칙

지속 가능한 1인 운영을 위한 핵심 원칙들. Claude는 이 원칙을 위배하는 제안을 하지 말 것.

## 원칙

### 1. 단일 진실 원천 (SSoT)

각 정보는 한 곳에만 산다. 동기화·복제는 자동화로만. 수동 복사 금지.

- 블로그 초안: `02.projects/factory9-pipeline/blog-queue/`만
- 결정 기록: `05.claude-context/evolution-log.md`만
- 세션 로그: `02.projects/sessions/`만

### 2. 추가만, 삭제는 명시적으로

`04.archive/`로 옮기는 것은 OK. 영구 삭제는 사용자가 직접만.

### 3. 비용을 측정하지 않으면 통제할 수 없다

모든 API 호출은 `_logs/cost-log.jsonl`에 기록. 매 세션 종료 시 누적 비용 표시.

### 4. 가드레일은 코드에, 정책은 문서에

- 재작성 1회 한계 → `blog_agent_pipeline.py` 코드에 박힘
- 품질 기준 → `agents/quality-standards/blog.md` 문서로
- 둘을 섞지 말 것

### 5. 새 기능은 마찰점에서만

이론적으로 멋진 기능 추가 금지. 실제 사용하면서 마찰을 느낀 후에만 추가.

### 6. 세션 컨텍스트는 자산이다

매 세션 시작에 5분 설명 = 1년에 30시간 손실. `current-state.md` 갱신을 의무화.

### 7. 사용자의 영역과 자동화의 영역을 명확히 구분

| 자동화 가능 | 사용자만 |
|---|---|
| 블로그 초안 생성 | 블로그 발행 결정 |
| QA 점수 산출 | 품질 기준 정의 |
| 비용 집계 | 예산 한계 결정 |
| 세션 로그 작성 | 비전·우선순위 |
